# Telegram-бот на базе Gemini

Бот принимает текст и изображения в Telegram, отвечает через Google Gemini API.
История диалога хранится в SQLite (контекст не теряется при перезапуске),
ответы выводятся в режиме стриминга — текст появляется по мере генерации.

Использует актуальный SDK **google-genai** (старый `google-generativeai` устарел).

## Возможности

- 💬 Ответы на текст через Gemini с учётом контекста диалога
- 🖼️ Поддержка изображений — отправь фото с подписью-вопросом
- 💾 Хранение истории в SQLite — переживает перезапуск бота
- ⌨️ Стриминг ответов — текст печатается постепенно
- Команды: `/start`, `/reset`, `/help`

## Что понадобится

1. **Токен Telegram-бота** — напиши [@BotFather](https://t.me/BotFather), команда `/newbot`.
2. **Ключ Gemini API** — https://aistudio.google.com/apikey (бесплатно, без карты).
3. **Python 3.10+**

## О бесплатности Gemini

- Модели семейства **Flash / Flash-Lite** бесплатны (по умолчанию `gemini-2.5-flash`).
- Лимиты бесплатного тарифа: ~1500 запросов в день, 15 запросов в минуту.
- ⚠️ На бесплатном тарифе Google может использовать твои запросы для обучения
  моделей — не отправляй конфиденциальные данные.
- ⚠️ Если включишь биллинг на проекте — бесплатный тариф на нём пропадёт,
  всё станет платным. Держи тест и прод в разных проектах.
- Pro-модели с апреля 2026 платные. Для бота их и не нужно — Flash хватает.

## Установка

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Создать .env из примера и вписать свои ключи
cp .env.example .env
# открой .env и подставь TELEGRAM_TOKEN и GEMINI_API_KEY

# 3. Запустить
python bot.py
```

При первом запуске рядом с ботом создастся файл `bot.db` — база с историей.

## Файлы проекта

- `bot.py` — основной код бота (хендлеры, стриминг, картинки)
- `db.py` — работа с SQLite (хранение и выборка истории)
- `requirements.txt` — зависимости
- `.env.example` — шаблон переменных окружения
- `.gitignore` — чтобы `.env` и `bot.db` не попали в Git
- `bot.db` — база данных (создаётся автоматически)

## Команды бота

- `/start` — начать, очистить историю
- `/reset` — очистить контекст диалога
- `/help` — справка

## Как это работает

- Каждое сообщение (текст или картинка) сохраняется в SQLite с ролью
  `user` / `model` (формат Gemini).
- При запросе из базы берутся последние 20 сообщений и собираются в `contents`.
- Системный промпт передаётся через `GenerateContentConfig(system_instruction=...)`.
- Ответ запрашивается через `client.aio.models.generate_content_stream(...)`,
  и сообщение в Telegram редактируется каждые ~0.7 сек по мере генерации.
- Изображение кодируется в base64, при запросе превращается в `types.Part`
  и отправляется модели (Flash умеет работать с картинками).

## Безопасность

- Токены и ключи держи только в `.env` — не вставляй их в код и не отправляй
  никому. Если ключ куда-то утёк — сразу отзови и создай новый.
- `.env` и `bot.db` уже добавлены в `.gitignore`.
